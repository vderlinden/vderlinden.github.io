Vraag & antwoord

  1. Begrippen

    • TCP

      TCP staat voor Transmission Control Protocol.

      TCP is een communicatiestandaard voor het uitwisselen van berichten (data).

    • IP-adres

      IP staat voor Internet Protocol.

      Het IP-adres kan gezien worden als de postcode en huisnummer van jouw woning, maar dan van je apparaat dat op het internet is aagesloten.

    • DNS

      DNS staat voor Domain Name System.

      DNS is en systeem(en protocol) dat IP-adressen vertaalt (lees: opzoeken) naar een domein (en omgekeerd).

    • Router

      Een router zorgt er voor dat een computer binnen een netwerk kan verbinden met een ander netwerk zoals het Internet.

    • Webserver

      Een webserver is een workstation met software om via een netwerk HTTP requests te verwerken en de opgevraagdedocumenten naar een client te sturen.

    • HTML

      HTML staat voor Hyper Text Markup Language.

      HTML is de standaard opmaaktaal voor het maken van webpagina's.

    • HTML tag

      Een HTML tag is een element in een bestand. Deze elementen vertellen de browser hoe de inhoud moet worden weergegeven (bijvoorbeeld een tabel of een plaatje). HTML-elementen labelen stukjes inhoud zoals "dit is een kop", "dit is een alinea", "dit is een link", enz.

  2. Hoe weet een browser wat voor soort bestand door de server wordt geretourneerd?

    De DOCTYPE tag bovenaan de pagina geeft een declaratie van het type document. Voor HTML5 is dat 'html'.

  3. Wat is het doel van keywords bij een website? Wie of wat gebruikt deze?

    Keywords geeft een zoekmachine meer informatie over de inhoud van de pagina.

  4. Wat is het doel van een description? Waar zal deze, wanneer je je site publiceert, te zien zijn?

    Een omschrijving geeft een zoekmachine een korte omschrijving/samenvatting van de inhoud van depagina. zoekmachines als Google tonen deze omschrijving in de zoekresultaten.

  5. Klik in je browser met je rechter muisknop op je pagina, en kies ‘Paginabron weergeven’ (Chrome: Ctrl+U). Wat zie je?

    Je ziet dan exact de code die je zojuist hebt geschreven.

  6. Klik op dezelfde manier en kies ‘Element inspecteren’ (Chrome: Ctrl+Shift+I). Beweeg met je muis over het scherm. Wat zie je? Wat is het nut hiervan?

    Je ziet dan de Inspector. Dit is een handige tool om de code (straks ook logic) van je website te bestuderen en debuggen. Met de element inspector kun je per element direct zien welke stijling is toegepast.

  8. Voeg een stuk tekst als alinea aan je pagina toe. Laat hierin zien wat de uitwerking is van aan aantal tags voor basisopmaak: vet, cursief, met nadruk, onderstreept.

    Tekst op een webpagina kan verschillende opmaak hebben:

    Denk aan vet gedrukte tekst door middel van de <b> tag. Gebruik <strong> als screanreaders er de nadruk op moeten leggen. Als het mogelijk is, gebruik dan eerst CSS!

    Denk aan schuingedrukte tekst door middel van de lt;i> tag. Doe dit met de <em> tag als screanreaders er de klemtoon op moeten leggen.

    Denk aan onderstreepte tekst door middel van de <u> tag.
